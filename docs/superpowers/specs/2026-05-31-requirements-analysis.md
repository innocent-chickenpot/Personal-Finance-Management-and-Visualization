# 个人财务管理与可视化系统 — 需求分析文档

## 1. 项目概述

### 1.1 项目名称
个人财务管理与可视化系统

### 1.2 项目简介
一个多用户 Web 端个人财务管理平台，帮助用户记录日常收支、管理多个支付渠道、通过可视化图表了解消费结构，并借助 AI 大模型提供消费行为分析和存钱建议。

### 1.3 目标用户
有个人/家庭记账需求，希望了解自身消费习惯、改善财务状况的普通用户。

### 1.4 核心价值
- 告别手工记账的繁琐，提供便捷的收支录入和导入
- 通过图表直观看到钱花在哪里、收入来自哪里
- AI 分析消费模式，给出可操作的省钱建议
- 多支付渠道统一管理，清楚掌握总资产状况

## 4. 用户故事

### US-1：注册与登录
> 作为一个新用户，我希望用邮箱注册一个账号，然后用密码登录，这样我的记账数据只有我能看到。

**验收条件：**
- 输入邮箱和密码完成注册，自动登录
- 输入正确邮箱密码登录成功，错误时提示
- 未登录访问任何页面自动跳转登录页
- 登录后可退出

### US-2：记录一笔支出
> 作为一个用户，我希望在消费后快速录入一笔记录，标注花了多少钱、属于哪类消费、通过什么渠道支付的，这样我能记住每一笔开销。

**验收条件：**
- 填写金额、选择分类、选择支付渠道、选择日期、可选备注，提交后保存
- 必填项为空时给出提示
- 提交成功跳转到记录列表，能看到刚添加的记录

### US-3：查看消费统计
> 作为一个用户，我希望能看到我的钱花在哪些地方，一眼看清各分类的占比。

**验收条件：**
- 首页显示总收入、总支出、余额三张卡片
- 显示支出分类饼图，各分类占比清晰可见
- 数据与录入的记录一致

### US-4：按时间查看账单
> 作为一个用户，我希望筛选某个月或某个时间段的收支记录，了解这段时间的财务状况。

**验收条件：**
- 选择日期范围后，列表只显示该范围内的记录
- 统计数据同步更新

### US-5：管理自己的分类
> 作为一个用户，我希望自定义收支分类，因为我的消费习惯可能和系统预设的不完全一致。

**验收条件：**
- 可以新增自定义支出/收入分类
- 可以重命名或删除自己创建的分类
- 系统预设分类不可删除
- 已使用的分类删除时给出提示

### US-6：导入账单
> 作为一个用户，我希望把支付宝/微信/银行导出的账单文件（CSV/Excel）直接导入系统，不用手动一条条录入。

**验收条件：**
- 上传 CSV 或 Excel 文件
- 系统解析并展示预览
- 确认后导入到记录列表

### US-7：预算控制
> 作为一个用户，我希望给"餐饮"设置每月 2000 的预算，当快花超或已经花超时，系统提醒我。

**验收条件：**
- 设置某分类的月度预算金额
- 当月消费超出预算时，页面显示警告

### US-8：AI 消费分析
> 作为一个用户，我希望 AI 帮我分析我的账单，告诉我哪些消费不合理，以及如何更好地存钱。

**验收条件：**
- 点击生成分析报告，AI 输出消费习惯总结和高消费日
- AI 给出针对性的存钱建议
- 导入账单时 AI 自动对条目进行分类

## 7. 约束与假设

### 约束
- 后端先部署在本地，后续考虑云端
- 数据库使用 MySQL
- AI 接入 DeepSeek API
- 当前版本不包含管理员角色

### 假设
- 用户具备基本的电脑操作能力
- 用户有稳定的网络连接（云端部署后）
- 短信验证码服务可用（迭代 2）

## 8. 范围边界

### 本期包含
- Web 端个人财务管理
- 多用户各自独立账本
- 手动记账 + 文件导入
- 图表可视化 + AI 分析

### 本期不包含
- 移动端原生 App
- 银行/支付宝/微信 API 自动同步
- 多人共享账本（家庭账本）
- 发票管理
- 理财/投资组合管理
- 多币种支持
