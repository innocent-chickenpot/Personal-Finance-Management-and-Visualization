## 二、模块详细设计

### 1. 顶部问候区

```
┌──────────────────────────────────────────────────────────┐
│  🌞  早安，壮壮！                 📅 2026年6月1日  星期一   │
│      今天是好好管理财务的第 15 天                            │
└──────────────────────────────────────────────────────────┘
```

- **时间判断**（本地时间）：
  - 0:00–5:59 → 🌙 夜深了
  - 6:00–8:59 → 🌅 早上好
  - 9:00–11:59 → ☀️ 上午好
  - 12:00–13:59 → 🌞 中午好
  - 14:00–17:59 → 🌤️ 下午好
  - 18:00–22:59 → 🌆 晚上好
  - 23:00–23:59 → 🌙 夜深了
- **用户名**：从 `auth.user.email` 取 `@` 前的部分作为昵称显示
- **"第X天"**：`今天 - user.createdAt` 的天数差，天数用 `var(--color-peach)` 蜜桃粉高亮
- **右侧**：完整日期 `YYYY年M月D日 星期X`
- **字体**：保持 ZCOOL KuaiLe 圆体

### 2. 四个指标卡片

| 卡片 | 数据来源 | 配色 |
|---|---|---|
| 🌸 本月收入 | 当月 `type=income` 合计 | 牛油果绿 `#6A8F5E` |
| 💐 本月支出 | 当月 `type=expense` 合计 | 蜜桃粉 `#C08080` |
| 🍀 结余 | 本月收入 − 本月支出 | 正数绿 / 负数粉 |
| 💰 储蓄率 | (结余 / 本月收入) × 100% | 焦糖棕 |

- 收入为0时储蓄率显示 `0.0%`
- 沿用现有贴纸卡片样式：24px 圆角、柔和阴影、hover 上浮

### 3. 收支趋势

- **图表类型**：柱状图，每月两根柱子并排
  - 收入柱：`#DDE8CF`（牛油果绿）
  - 支出柱：`#F7D6D6`（蜜桃粉）
- **横轴**：1月–12月（当年全部月份，无数据的月份柱高为0）
- **纵轴**：金额，自动刻度
- **柱子样式**：圆角顶部 `borderRadius: [8, 8, 0, 0]`
- **tooltip**：奶油白底色 `#FFFDF9`，焦糖棕文字
- **新增 API**：`GET /api/stats/monthly-trend?year=2026`
  - 返回：`[{ month: 1, income: 8500, expense: 3200 }, ...]`

### 4. 消费分类占比

- 保留现有 ECharts 环图（`radius: ['48%', '74%']`）
- 现有 API `GET /api/stats/by-category` 不变
- 配色沿用 `CREAM_COLORS` 数组

### 5. 储蓄罐

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│   🐶 盖子     │   │   🐶 盖子     │   │   ✚ 新建     │
│  ╭──────────╮ │   │  ╭──────────╮ │   │  ┌ ─ ─ ─ ┐ │
│  │  💰💰💰  │ │   │  │  💰💰    │ │   │  │  +   │ │
│  │  💰💰💰  │ │   │  │  💰💰    │ │   │  └ ─ ─ ─ ┘ │
│  │  💰💰    │ │   │  │          │ │   │              │
│  ╰──────────╯ │   │  ╰──────────╯ │   │              │
│   🏖️ 旅游基金 │   │   🎮 游戏机    │   └──────────────┘
│   ¥5,000/8K  │   │   ¥800/3K     │
│   62.5%      │   │   26.7%        │
└──────────────┘   └──────────────┘
```

- **罐子视觉**：CSS 绘制玻璃罐（渐变透明 + 圆角底部），奶油小狗趴在罐口
  - 罐内硬币堆叠高度 = 完成率百分比
  - 罐子满了 → 金币满格 + 小狗表情开心 + 显示"已完成"
- **横向排列**，超出 `overflow-x: auto`
- **"+ 新建"按钮**：柔和虚线边框卡片
- **存入操作**：
  1. 点击罐子 → 弹窗输入金额
  2. 自动创建 `支出` 交易记录（分类：系统预设"转账至储蓄罐"）
  3. 罐子 `savedAmount` 更新
- **新增数据表 `savings_jar`**：

```sql
id          INT AUTO_INCREMENT PRIMARY KEY
userId      INT NOT NULL
name        VARCHAR(50) NOT NULL
targetAmount DECIMAL(12,2) NOT NULL
savedAmount  DECIMAL(12,2) DEFAULT 0
createdAt   DATETIME DEFAULT NOW()
```

- **新增 API**：
  - `GET /api/savings-jars` — 当前用户的罐子列表
  - `POST /api/savings-jars` — 创建罐子 `{ name, targetAmount }`
  - `PUT /api/savings-jars/:id` — 编辑罐子
  - `DELETE /api/savings-jars/:id` — 删除罐子
  - `POST /api/savings-jars/:id/deposit` — 存入 `{ amount }`，自动创建支出记录
- **新增分类**：seed 脚本中添加预设分类"转账至储蓄罐"（type=expense）

## 四、新增 API 详情

### GET /api/stats/monthly-trend?year=2026

返回当年1-12月每月收入/支出合计：

```json
{
  "code": 200,
  "data": [
    { "month": 1, "income": 8500, "expense": 3200 },
    { "month": 2, "income": 8500, "expense": 2800 }
  ]
}
```

### GET /api/savings-jars

```json
{
  "code": 200,
  "data": [
    { "id": 1, "name": "旅游基金", "targetAmount": 8000, "savedAmount": 5000, "createdAt": "..." }
  ]
}
```

### POST /api/savings-jars

```json
// Request
{ "name": "旅游基金", "targetAmount": 8000 }
// Response
{ "code": 200, "data": { "id": 1, ... } }
```

### POST /api/savings-jars/:id/deposit

```json
// Request
{ "amount": 1000 }
// Response
{ "code": 200, "data": { "id": 1, "savedAmount": 6000 } }
// 副作用：自动创建一条 type=expense, category="转账至储蓄罐" 的交易记录
```

## 五、禁止项（保持奶油治愈系）

- 不使用科技蓝、Material Design、深色模式
- 不使用高饱和颜色、锐利直角、强烈渐变
- 不使用 side-stripe borders
- 不使用 gradient text
- 不使用 hero-metric 模板
- 不使用 glassmorphism（储蓄罐的玻璃质感为功能性，非装饰性）
